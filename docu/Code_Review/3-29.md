# Code Review 3-29

## Review 范围
- `ArticleService`（本轮新增/修订功能）
- `UserService`（含“查询自己发布文章”）
- `ArticlesController`、`AuthController`
- 相关 Repository / DTO

## 本轮核心修复项
1. 路由参数绑定统一  
- 统一了路由模板与参数名（`articleId`），避免模型绑定失败。

2. 点赞存在性判断修复  
- 修复了点赞查询“总是 true”的逻辑错误。  
- 统一了 `userId, articleId` 参数语义，降低后续误用风险。

3. 发布文章越权修复  
- 移除前端可控作者身份来源。  
- 发布接口改为使用 JWT 中的用户 ID 作为作者 ID。

4. 文章详情空引用修复  
- 调整为空先判空，再构造 `DisplayArticleDTO`，避免空引用异常。

5. 点赞并发一致性改造  
- 点赞/取消点赞改为事务执行。  
- 失败回滚，避免“点赞记录与 LikeCount 不一致”。

6. 编辑文章事务化  
- 文章内容更新 + 标签删增纳入同一事务。  
- 失败回滚，避免半成功状态。  
- 补充 `TagIds` 判空与事务提交逻辑。

7. 封面字段一致性修复  
- `PublishArticleDTO` 增加 `CoverUrl`。  
- 发布文章时写入 `CoverUrl`，消除实体非空字段落库风险。

8. 我的文章列表分页优化  
- 接口改为 `FromQuery` 接收分页参数。  
- 增加边界校验（页码/每页数量）。  
- 引入 `PagedResult<T>` 返回分页数据与总数。

9. 日志补齐与审计增强  
- `ArticleService` 主要异常路径补齐 `LogError`/`LogWarning`。  
- `AuthController` 与 `ArticlesController` 补充未认证分支日志。  
- `ArticleRepository` 日志类型修正为 `ILogger<ArticleRepository>`。
