## 项目名称：**个人博客管理系统**

### 一、项目背景

本项目旨在开发一个功能完善的个人博客系统，包含**前台内容展示**与**后台管理**两大模块。前端使用 Vue 3 构建单页应用，后端基于 [ASP.NET](https://asp.net/) Core 提供 RESTful API，数据库采用 SQL Server。通过该项目可以系统学习前后端分离开发模式、认证授权、数据库设计、文件上传、数据可视化等核心技能。

------

### 三、功能需求

#### 1. 用户模块

| 功能点       | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| 用户注册     | 提供用户名、邮箱、密码注册；密码需加密存储（BCrypt 或 PBKDF2）。 |
| 用户登录     | 支持用户名/邮箱 + 密码登录，成功后返回 JWT 令牌；前端存储令牌并自动携带在请求头中。 |
| 个人资料管理 | 登录后可查看/修改个人信息（昵称、头像、个人简介）。头像支持图片上传，后端保存文件并返回 URL。 |
| 用户角色     | 内置两种角色：**普通用户**（可评论、点赞）和**管理员**（可管理文章、分类、标签、评论）。初始化时可通过数据库种子添加一个管理员账号。 |

#### 2. 文章模块（核心）

| 功能点         | 描述                                                         |
| :------------- | :----------------------------------------------------------- |
| 文章列表       | 前台支持分页展示文章列表，按发布时间倒序排列；每篇文章显示标题、摘要、封面图、发布时间、阅读量、点赞数、所属分类及标签。 |
| 文章搜索与筛选 | 支持按标题关键字模糊搜索；可按分类筛选文章；可同时按标签筛选。 |
| 文章详情       | 展示文章完整内容（支持 Markdown 渲染）、作者信息、发布时间、阅读量实时增加（注意防止刷量）。 |
| 文章发布/编辑  | **仅管理员可操作**。使用 Markdown 编辑器（如 Vditor 或 ByteMD）编写文章；支持设置标题、摘要、封面图、分类、标签（多选）。编辑时保留原数据。 |
| 文章删除       | 仅管理员可删除文章，删除时需同时删除文章-标签关联关系；评论记录可选择级联删除或保留为“文章已删除”状态。 |
| 文章置顶/推荐  | 管理员可将文章设为“置顶”（列表优先显示）或“推荐”（用于首页轮播或精选）。 |

#### 3. 分类与标签模块

|     功能点      | 描述                                                         |
| :-------------: | :----------------------------------------------------------- |
|    分类管理     | 管理员可增删改分类（名称、描述、排序值）。一篇文章只属于一个分类。 |
|    标签管理     | 管理员可增删改标签（名称）。一篇文章可关联多个标签（多对多关系）。 |
| 前台分类/标签页 | 点击分类或标签时，展示该分类/标签下的文章列表，同样支持分页和搜索。 |

#### 4. 评论模块

|  功能点  | 描述                                                         |
| :------: | :----------------------------------------------------------- |
| 发表评论 | 任何登录用户均可对文章发表评论，内容为纯文本，需限制长度（≤500字）。 |
| 评论列表 | 文章详情页下方展示该文章的所有评论，按发布时间正序排列；显示评论者昵称、头像、评论时间、内容。 |
| 删除评论 | 普通用户可删除自己的评论；管理员可删除任何评论。             |
| 评论审核 | 扩展功能：管理员可开启评论审核模式，新评论需审核通过后才在前台显示。 |

#### 5. 统计与仪表盘

|    功能点    | 描述                                                         |
| :----------: | :----------------------------------------------------------- |
| 管理后台首页 | 展示统计数据：总文章数、总评论数、总分类数、总标签数；近 7 天文章发布趋势（柱状图）；文章阅读量排行 Top5（饼图或列表）。 |
|  个人仪表盘  | 普通用户登录后可查看自己发表过的评论数量、点赞过的文章列表等。 |

#### 6. 系统功能

| 功能点       | 描述                                                         |
| :----------- | :----------------------------------------------------------- |
| 全局异常处理 | 后端统一返回格式（如 `{ success: false, message: string, data: null }`），前端统一拦截并提示。 |
| 请求日志     | 使用中间件记录每个请求的路径、方法、耗时、IP 地址等信息。    |
| API 文档     | 通过 Swagger 自动生成 API 文档，方便前端调试和接口测试。     |
| 分页统一规范 | 所有列表接口均支持 `pageIndex` 和 `pageSize` 参数，返回数据包含总记录数、总页数等分页信息。 |

### 六、前端页面结构（Vue 路由规划）

- **前台展示模块**
  - `/` 首页：展示置顶文章列表、推荐文章列表、最新文章列表（分页）。
  - `/articles/:id` 文章详情页：文章内容、评论区。
  - `/categories/:id` 分类文章列表页。
  - `/tags/:id` 标签文章列表页。
  - `/search` 搜索结果页。
  - `/login` / `/register` 登录/注册页。
- **后台管理模块**（需登录且角色为管理员才可访问）
  - `/admin/dashboard` 仪表盘（数据统计图表）。
  - `/admin/articles` 文章管理（列表、新建、编辑、删除）。
  - `/admin/categories` 分类管理（增删改查）。
  - `/admin/tags` 标签管理（增删改查）。
  - `/admin/comments` 评论管理（查看所有评论、删除、审核）。
  - `/admin/profile` 个人资料修改（普通用户和管理员共用）。
- **公共组件**
  - 导航栏（包含登录状态、用户头像下拉菜单、后台入口等）。
  - Markdown 编辑器组件（封装为公共组件）。
  - 分页组件。

### 七、其他功能需求

1. **安全性**
   - 密码存储必须哈希加盐。
   - JWT 令牌有效期建议 7 天，支持刷新令牌。
   - 所有 API 需防范 SQL 注入（使用 EF Core 参数化查询）。
   - 文件上传限制类型（仅图片）、大小（≤ 2MB），并对文件名进行随机化处理。
2. **性能**
   - 文章列表和详情接口应包含必要的索引。
   - 对于阅读量频繁更新的场景，可考虑使用缓存或异步更新（如先更新内存，定期批量写入数据库），为了防止刷文章阅读量，至少阅读两分钟以上才加一次阅读量。
   - 前端实现路由懒加载。
3. **用户体验**
   - 所有异步请求需有 Loading 状态提示。
   - 表单提交后显示成功/失败消息，并自动跳转或刷新列表。
   - 移动端响应式布局（至少保证基本可用）。
4. **代码规范**
   - 后端遵循 RESTful 设计原则，使用 DTO 与实体分离，借助 AutoMapper 转换。
   - 前端使用组合式 API (Composition API)，状态管理按模块拆分。

### 八、扩展功能

1. **文章点赞**：用户可对文章点赞，一篇文章一个用户只能点赞一次，记录点赞关系。
2. **嵌套评论**：支持对已有评论的回复，形成多级讨论。
3. **站内邮件通知**：当用户评论被回复时，发送站内邮件提醒。
4. **RSS 订阅**：提供文章列表的 RSS 订阅源。
5. **访问统计**：使用第三方工具（如 Google Analytics）或自研 PV/UV 统计。
6. **国际化**：前端支持中英文切换。
